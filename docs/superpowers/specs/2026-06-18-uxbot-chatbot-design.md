# inswave UX 담당 챗봇 (UXBot) 디자인 스펙

- **작성일**: 2026-06-18
- **대상 페이지**: `index-immersive.html` (inswave 스크롤 내러티브 진입화면)

## 1. 개요

inswave 진입화면에 **UX 컨설팅팀 문의를 응대하는 UI/UX 담당 챗봇(UXBot)** 을 추가한다.
사용자가 UX 컨설팅 서비스에 대해 묻는 모든 문의(서비스 소개·진행 프로세스·기간/비용·자주 묻는 질문)에 답한다.

최종 목표는 **Claude API(`claude-opus-4-8`) 연동 LLM 챗봇**이지만, 현재 프로젝트는 백엔드 없는
프론트 단독 정적 파일 구조다. 브라우저에서 API 키를 직접 다루면 키가 노출되는 보안 문제가 있으므로,
**이번 단계는 LLM을 직접 연결하지 않고 "목업(mock) 응답"으로 구현**한다. 단, LLM 호출을 단일
어댑터 함수로 추상화해 **나중에 어댑터 본문만 교체하면 실제 Claude API로 전환**되도록 설계한다.

## 2. 범위

### 이번 단계 (In scope)
- `index-immersive.html` 내에 챗 위젯 UI (플로팅 버튼 → 챗 패널)
- UX팀 지식 기반 **목업 응답 엔진** (의도/키워드 매칭 → 정해진 답변, 타이핑 스트리밍 효과)
- LLM 어댑터 인터페이스 `askUXBot(message, history)` — 지금은 mock 구현
- 범위 밖 질문 가드레일(정중히 "문의하기"로 유도)
- 접근성(키보드, 포커스, 대비, `prefers-reduced-motion`) 및 브랜드 톤 일치

### 비범위 (Out of scope / YAGNI)
- 실제 Claude API 호출, API 키 처리, 백엔드 프록시
- 대화 영속화/DB, 사용자 인증, 리드 접수/상담예약 폼
- 다국어

## 3. 아키텍처

```
[플로팅 버튼] → [챗 패널]
                   │  사용자 입력
                   ▼
            askUXBot(message, history)   ← 단일 어댑터 (교체 지점)
                   │  (현재: mock 모드)
                   ▼
        [의도 매칭 → 지식 베이스 답변] → 스트리밍 출력(타이핑 효과)
```

- **단일 교체 지점**: `askUXBot()` 는 비동기이며, 응답 청크를 콜백/제너레이터로 흘려보내는
  스트리밍 인터페이스를 갖는다. 추후 이 함수 본문만 Claude `/v1/messages` 스트리밍 호출로 바꾸면
  UI 변경 없이 LLM 전환된다. (운영 시에는 키 노출 방지를 위해 백엔드 프록시 경유 권장.)

## 4. 컴포넌트

### 4.1 챗 위젯 UI
- 우하단 플로팅 버튼(기존 "상담 문의" 버튼을 토글로 전환): 클릭 시 챗 패널 open/close
- 챗 패널: 헤더(`UX 담당봇` + 닫기) · 메시지 리스트(사용자/봇 말풍선) · 입력창 + 전송 버튼
- 첫 진입 시 환영 메시지 + 추천 질문 칩(빠른 시작): 예) "UX 컨설팅 절차가 궁금해요", "비용은 얼마인가요"
- 봇 응답 중 타이핑 인디케이터, 스트리밍 텍스트 출력
- 다크+시안 브랜드 톤, 반응형, `prefers-reduced-motion` 시 애니메이션 축소

### 4.2 지식 베이스 (목업 응답 데이터)
UX 컨설팅팀 문의에 답할 수 있는 항목 (값은 합리적 더미, 실제 수치 확정 전까지 "범위/예시"로 표기):
- **서비스 소개**: UX 컨설팅이 다루는 영역(사용성 진단, UX 전략, UI 설계, 디자인 시스템 등)
- **진행 프로세스**: 단계별 흐름(문의 → 진단/리서치 → 설계 → 검증 → 핸드오프)
- **기간/비용**: 프로젝트 규모별 기간·비용 "범위" 안내(확정 견적은 문의 유도)
- **FAQ**: 자주 묻는 질문 N개(원격 진행 가능 여부, 산출물, 협업 방식 등)
- **가드레일/폴백**: 매칭 실패·범위 밖 → "정확한 답변을 위해 문의하기로 연결해 드릴게요" + CTA

### 4.3 어댑터 `askUXBot(message, history)`
- 입력: 사용자 메시지, 이전 대화 배열
- 동작(mock): 메시지를 정규화 → 키워드/의도 매칭 → 해당 지식 항목 텍스트 반환
- 출력: 응답 텍스트를 청크 단위로 흘려보냄(스트리밍 UX 유지)
- 추후 live 모드: 동일 시그니처로 Claude `claude-opus-4-8` 스트리밍 호출 + UX팀 시스템 프롬프트

## 5. 산출물
- 수정: `index-immersive.html` (챗 위젯 UI + 목업 엔진 인라인)
- (지식 베이스는 같은 파일 내 JS 데이터 구조로 인라인)

## 6. 향후 (LLM 전환 시 메모)
- 시스템 프롬프트: UX 컨설팅팀 페르소나 + 지식 + 가드레일(범위 밖 차단, 모르는 건 지어내지 않기)
- 모델 `claude-opus-4-8`, `/v1/messages` 스트리밍, 멀티턴, 시스템 프롬프트 prompt caching
- 키 노출 방지를 위해 **백엔드 프록시 경유** (프론트 직접 호출은 데모 한정)
